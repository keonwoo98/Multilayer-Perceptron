## 2. 클래스 설계

### DenseLayer (한 층 = STEP 2/4/5)
```
상태:  W (뉴런수, 입력수), b (뉴런수,), activation, cache(forward 중간값)

forward(입력):                          # STEP2 예측
    z = W @ 입력 + b ;  a = 활성화(z)
    cache에 입력,z,a 저장 ;  return a

backward(오차):                         # STEP5 기울기
    dW = 오차 ⊗ 입력 ;  db = 오차
    앞층_오차 = (W.T @ 오차) × sigmoid미분
    return 앞층_오차

update(lr):                             # STEP4
    W -= lr × dW ;  b -= lr × db
```

### Network (층들을 묶음 = 전체 모델)
```
상태:  layers = [DenseLayer, ...]

forward(입력):   for 층 in layers: 입력 = 층.forward(입력); return 입력
backward(예측,정답):
    오차 = 예측 - 정답                    # STEP5 파트2의 마법
    for 층 in reversed(layers): 오차 = 층.backward(오차)   # 거꾸로!
fit(train, valid, epochs, lr, batch_size):  # 학습 루프
predict(입력):  return forward(입력)
```

핵심: reversed(layers)가 backprop의 "출력→입력" 방향.

## 3. 데이터 흐름 (한 epoch)

```
epochs번 반복:
  1 epoch = 전체 train을 batch_size씩 잘라 처리 (455÷8 ≈ 57배치)

  배치 1개(8샘플) 처리:
    ① forward          → 8개 예측 (STEP2)
    ② loss 계산        → cross-entropy (STEP3)
    ③ backward         → 오차=예측-정답, 거꾸로 전파, dW/db (STEP5)
    ④ update           → W -= lr×dW (STEP4)

  epoch 끝:
    - train_loss, val_loss, accuracy 계산 & history 저장
    - 출력: "epoch 40/84 - loss: 0.0749 - val_loss: 0.0404"  (PDF 형식)
```

- 배치 8개를 행렬에 쌓아 한 번에 처리(벡터화). 입력 (8,30) @ W로 8개 동시 계산.
- validation은 학습에 안 씀(평가만). train_loss는 주는데 val_loss 정체/상승 = 과적합.

## 4. 전처리 & 모델 저장/로드

```
data.csv ─split.py→ data_train.csv + data_valid.csv
         ─train.py→ models/model.npz (가중치+구조+μ,σ)
         ─predict.py→ 예측 + binary cross-entropy 평가
```

### ★ data leakage 함정 (중요)
표준화 = (값 - 평균μ) / 표준편차σ
- μ, σ는 **train 데이터로만** 계산
- valid/predict도 **train의 μ, σ를 그대로** 적용 (각자 평균 쓰면 정보 누출=cheating)
- 그래서 μ, σ를 모델과 함께 저장 → predict에서 동일 변환 재사용

### 저장 내용 (model.npz)
- 각 층의 W, b (학습 결과)
- 네트워크 구조 (층 크기, 활성화 함수)
- 정규화 파라미터 μ, σ

### predict.py 평가 공식 (PDF)
```
E = -(1/N) Σ [y·log(p) + (1-y)·log(1-p)]    # binary cross-entropy
```

## 5. 구현 순서 (단계별 빌드업)

원칙: 작은 조각 → 즉시 검증(docs의 손계산값과 대조) → 다음 조각.

```
[1] activations.py   검증: sigmoid(8.0)=0.9997, softmax([1,2])=[0.27,0.73]
[2] preprocessing.py 검증: 정규화 후 평균≈0, 표준편차≈1
[3] split.py         검증: train+valid 행수 합 = 569
[4] layer.py(forward) 검증: 3뉴런 예제 [8.0,-1.6,0.8] (docs STEP2)
[5] network.forward  검증: 출력 [확률,확률] 합=1
[6] losses.py        검증: -log(0.73)=0.31 (docs STEP3)
── 예측 절반 완료 / 학습 절반 시작 ──
[7] backward         검증: gradient checking (수치미분 ≈ backward) ★고비
[8] update + fit     검증: loss가 epoch마다 감소
[9] train.py         검증: PDF 형식 출력 + model.npz 저장
[10] predict.py      검증: validation 정확도 ~97%+
[11] 학습곡선 그래프  loss곡선 + accuracy곡선 (PDF 필수)
```

검증 정답지는 이미 docs에 있음(numpy로 검증된 값). 7단계가 고비 — gradient checking으로 확신 확보.

## 6. 보너스 계획 (mandatory 완성 후)

mandatory가 PERFECT한 뒤 구현. 확장이 쉽도록 미리 설계에 반영:

| 보너스 | 확장 지점 | 비고 |
|--------|-----------|------|
| Adam 등 optimizer | layer.update를 Optimizer 클래스로 분리 | SGD→Adam 교체 쉽게 |
| early stopping | fit의 epoch 루프에 val_loss 모니터링 | 개선 없으면 조기 종료 |
| 다중 metric | metrics.py에 precision/recall/F1 추가 | |
| 학습곡선 여러 개 한 그래프 | history 저장 구조 | 모델 비교용 |
| metric history | fit에서 epoch별 기록 | |
