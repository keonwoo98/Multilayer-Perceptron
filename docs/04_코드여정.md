## 1부: 데이터 준비

환자 1명 = data.csv의 한 줄:
```
842302, M, 17.99, 10.38, 122.8, 1001, 0.1184, ...
  id   정답  ←──────── feature 30개 ────────→
```

### split.py — 섞어서 나누기
569명을 무작위로 섞어 456명(train) / 113명(valid)으로 저장.
- `np.random.permutation(n)`: 0~568 번호를 무작위 순서로 → 섞기
- `np.random.seed(seed)`: 시작점 고정 → 매번 같은 분리 (재현성)
- 안 섞으면 앞쪽 M만 train에 몰림 → 그래서 섞는다

### srcs/preprocessing.py — 신경망이 먹을 형태로

**load_data**: 표(df)에서 "문제(X)"와 "답(y)"을 분리
```python
y = df["diagnosis"].values                       # 답:  'M'
X = df.drop(columns=["id","diagnosis"]).values   # 문제: feature 30개
```
왜 분리? 지도학습이라 "문제 풀게 하고 답과 비교"해야 하니까.

**standardize**: 스케일 맞추기 (area 1001 같은 큰 값 문제 해결)
```python
return (X - mean) / std
```
```
[17.99, 10.38, 122.8, 1001, ...]  →  [1.10, -2.07, 1.27, ...]
   raw (스케일 제각각)                 표준화 (평균0 기준)
```
- mean, std는 compute_stats가 train에서만 계산 (valid에도 train 값 재사용 = leakage 방지)

**one_hot**: 정답 'M'을 숫자 쌍으로
```python
'M' → [0, 1]      'B' → [1, 0]
```
예측(숫자 2개)과 같은 형식으로 만들어 비교 가능하게.

→ 이제 환자는 **입력 [1.10, -2.07, ...](30개)** 와 **정답 [0,1]** 로 변신 완료.

## 2부: 예측 (forward)

### srcs/network.py — Network.forward (28~32줄)
```python
def forward(self, x):
    for layer in self.layers:    # 층 3개를 순서대로
        x = layer.forward(x)     # 한 층 출력이 다음 입력
    return x
```
x가 층을 지날 때마다 변신:
```
[30개] → 은닉1 → [24개] → 은닉2 → [24개] → 출력 → [2개]
```

### srcs/layer.py — DenseLayer.forward (23~36줄)
```python
def forward(self, x):
    self.input = x                  # ★ 입력 기억 (backward가 씀)
    z = self.W @ x + self.b         # 가중합 (STEP 2)
    if self.activation == "sigmoid":
        return sigmoid(z)           # 은닉층: 0~1
    elif self.activation == "softmax":
        return softmax(z)           # 출력층: 확률(합=1)
```
- `self.W @ x + self.b`: STEP 2의 `z = W×입력 + b`. `@`는 행렬곱.
- 활성화 함수 씌우면 그 층 출력.
- **`self.input = x` 주목**: 지금 안 쓰지만 저장해둠. backward(3부)에서
  "이 층에 뭐가 들어왔지?"가 필요하기 때문. → forward를 먼저 해야 backward 가능.

학습 초반(랜덤 가중치): 입력 → forward → [0.499, 0.501] (반반, 아직 못 맞힘)
학습 후(좋은 가중치):   같은 입력 → forward → [0.02, 0.98] (악성 98%!)
같은 입력인데 가중치가 바뀌어 예측이 정답 쪽으로 이동. 그 "바꾸는 게" 3부.

## 3부: 학습 (핵심) — loss → backward → update

### srcs/network.py — fit의 한 샘플 처리 (74~77줄)
```python
for i in np.random.permutation(n):    # 매 epoch 샘플 순서 섞기
    pred = self.forward(X_train[i])   # ① 예측
    self.backward(pred, y_train[i])   # ② 기울기 구하기
    self.update(lr)                   # ③ 가중치 수정
```
딱 3줄. 이걸 456명 × epoch번 반복 = 학습.

### loss는 어디? (헷갈리기 쉬운 점)
위 3줄에 loss 함수 호출이 없다. 왜?
- loss = "얼마나 틀렸나" 점수판. backward = "그 loss를 줄이는 방향(기울기)".
- softmax+cross-entropy의 미분이 **그냥 (예측 - 정답)** 으로 단순해진다 (STEP 5-B 마법).
- 그래서 backward는 loss 함수를 안 거치고 `예측-정답`에서 바로 시작.
- losses.py는 fit의 epoch 끝 evaluate에서 "사람이 보려고" 계산할 때만 쓰임.

### ② backward — srcs/network.py (34~43줄)
```python
def backward(self, y_pred, y_true):
    delta = y_pred - y_true            # 출력층 오차 (시작점!)
    for layer in reversed(self.layers):  # 출력→은닉2→은닉1 (거꾸로!)
        delta = layer.backward(delta)
```
우리 환자(초반):
```
delta = [0.499, 0.501] - [0, 1] = [0.499, -0.499]
양성 자리 +0.499: "양성 확률 너무 높게 줬다"
악성 자리 -0.499: "악성 확률 부족하다"
```
- `reversed`: STEP 5의 "왜 거꾸로?" = loss가 출력 끝에서 나오니 거기서 거슬러옴.
- 잘 맞힐수록 delta가 0에 가까워짐 → 덜 고침 (자동 조절).
  예: 후반 [0.1,0.9]-[0,1]=[0.1,-0.1] (0에 가까움)

### 각 층의 backward — srcs/layer.py (38~51줄)
```python
def backward(self, delta):
    self.dW = np.outer(delta, self.input)   # ① 가중치 기울기
    self.db = delta                         # ② bias 기울기
    da_prev = self.W.T @ delta              # ③ 앞 층으로 오차 전달
    return da_prev * sigmoid_derivative(self.input)  # ④ sigmoid 미분
```
- **① `np.outer(delta, self.input)`**: "오차 × 입력" = 이 층 가중치 기울기(dW).
  - 입력이 컸다 → 출력에 크게 기여 → 책임 큼 → 많이 고쳐야
  - 오차가 컸다 → 많이 틀림 → 많이 고쳐야
  - `np.outer`: 벡터 둘로 행렬 만들기. delta(뉴런수)×input(입력수) = W와 같은 크기.
  - ★ 여기서 forward가 저장한 `self.input`을 꺼내 씀!
- **② db = delta**: bias는 입력이 1이라 오차 그대로.
- **③④ 앞 층 전파**: W.T로 오차를 거꾸로 보내고(`.T`=방향 뒤집기),
  앞 층 sigmoid 미분 a(1-a)을 곱함 (연쇄법칙). self.input이 앞 층 출력이라 재사용.
- 반환값 = 앞 층의 delta → 다음 반복으로.

backward 한 번 = 모든 층의 dW, db가 채워짐.

### ③ update — srcs/layer.py (53~56줄)
```python
def update(self, lr):
    self.W -= lr * self.dW    # STEP 4: 새 W = 옛 W - 학습률 × 기울기
    self.b -= lr * self.db
```
backward가 구해둔 self.dW를 학습률만큼 빼서 가중치 수정.
network.update(45~48줄)는 모든 층에 이걸 시킴.

### 역할 분담 (중요!)
```
forward:  self.input 저장   → backward가 쓸 입력
backward: self.dW 저장      → update가 쓸 기울기
update:   self.W 수정       → 저장된 dW로
```
그래서 순서가 항상 forward → backward → update.
`self.`로 저장해두고 다음 단계에서 꺼내 쓰는 흐름.

이 3줄을 456명 × 30 epoch 반복 → loss 0.57 → 0.035, 정확도 99%.

## 4부: 저장 · 로드 · 평가

### 저장 — train.py
```python
params = {"layer_sizes": ..., "mean": mean, "std": std}
for i, layer in enumerate(net.layers):
    params[f"W{i}"] = layer.W     # 학습된 가중치
    params[f"b{i}"] = layer.b
np.savez(MODEL_PATH, **params)
```
3가지 저장: 가중치(학습 결과) + layer_sizes(구조) + mean,std(정규화 기준).

### 로드 — predict.py (load_model)
```python
data = np.load(path)
net = Network(data["layer_sizes"])    # 같은 구조 빈 네트워크
for i, layer in enumerate(net.layers):
    layer.W = data[f"W{i}"]          # 저장된 가중치 덮어쓰기
    layer.b = data[f"b{i}"]
```
저장의 반대: 같은 모양 만들고 → 가중치 덮어쓰면 → 학습된 네트워크 복원.

### 평가 — predict.py
```python
X = standardize(X, mean, std)         # ★ 저장된 mean,std로 정규화
preds = np.array([net.forward(xi) for xi in X])
accuracy = np.mean(preds.argmax(axis=1) == y.argmax(axis=1))
bce = binary_cross_entropy(preds[:, 1], y[:, 1])
```
- 저장된 mean,std 재사용 (새로 평균 X). 가중치가 train 기준이라 입력도 같은 기준이어야.
- argmax: 더 큰 확률 쪽이 예측. 정답과 비교 → 정확도.
- binary_cross_entropy: PDF 지정 평가 공식.

### ★ 왜 mean,std 저장이 중요?
가중치는 "train 기준"으로 학습됨 → 입력도 똑같은 train 기준으로 변환해야 맞음.
predict에서 새 평균 쓰면 기준이 안 맞아 예측 엉망. 그래서 모델과 한 몸으로 저장.

```

4부 마무리: 저장(가중치+구조+mean,std) → 로드(복원) → 평가(BCE)
```
