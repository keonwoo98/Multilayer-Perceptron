# Multilayer Perceptron — 핵심 개념 정리

> 이 문서는 신경망 학습의 전체 흐름(STEP 1~5)을 정리한 것이다.
> 디펜스 전이나 까먹었을 때 이 문서 하나만 읽으면 큰 그림이 다시 떠오르도록 만들었다.
> 모든 숫자 예시는 numpy로 검증된 값이다.

---

## 0. 큰 그림 — 한 문장

**신경망 = "숫자 30개(세포 특징)를 넣으면 → 악성일 확률을 뱉어내는 거대한 수학 함수"**

이 함수 안에는 조절 가능한 손잡이(가중치, weight)가 수천 개 있다.
처음엔 손잡이가 랜덤이라 엉터리 답을 낸다.
**학습(training)** = 정답을 보면서 이 손잡이들을 조금씩 돌려 점점 정답에 가까워지게 만드는 것.

비유: 샤워기 온도 맞추기. 너무 뜨겁/차가움(오답) → 손잡이 조금 돌림(가중치 조정) → 온도 느낌(오차 확인) → 반복 → 적정 온도(정답) 수렴.

---

## 전체 학습 루프 (이게 곧 train 프로그램)

```
1. 데이터 정규화 + label을 one-hot으로 변환
2. 가중치 랜덤 초기화
3. epoch만큼 반복:
     a. feedforward로 예측        (STEP 2)
     b. loss 계산: 얼마나 틀렸나   (STEP 3)
     c. backpropagation: 각 가중치 기울기 구하기  (STEP 5)
     d. gradient descent: 가중치 업데이트         (STEP 4)
4. 학습된 모델(구조 + 가중치) 저장
```

핵심: **feedforward(앞으로 예측) + backpropagation(뒤로 기울기)** 의 왕복을 epoch만큼 반복.

## STEP 1 — 뉴런 1개 (퍼셉트론)

뉴런 하나가 하는 일은 딱 2단계.

```
1단계 (가중합):  z = (x1 × w1) + (x2 × w2) + ... + b
2단계 (활성화):  출력 = sigmoid(z)
```

- **x** = 입력값 (예: 세포 반지름)
- **w (가중치)** = "이 입력이 얼마나 중요한가" → 학습으로 찾아내는 값
- **b (bias, 편향)** = 기준선을 위아래로 옮기는 보정값
- **sigmoid** = 어떤 값이든 0~1 사이로 부드럽게 눌러주는 S자 함수

```
sigmoid(z) = 1 / (1 + e^(-z))
```

### 검증된 숫자 예시
입력 x1=18.0, x2=1000.0 / 가중치 w1=0.5, w2=0.002 / b=-3.0

```
z = 18.0×0.5 + 1000.0×0.002 + (-3.0) = 9.0 + 2.0 - 3.0 = 8.0
sigmoid(8.0) = 0.99966   → "악성일 확률 약 99.97%" (1에 가까우면 악성 쪽)
```

### sigmoid 감 잡기
| z      | sigmoid(z) | 의미              |
|--------|-----------|-------------------|
| -8.0   | 0.0003    | 거의 0 (양성 확신)  |
|  0.0   | 0.5       | 반반 (모르겠음)     |
|  8.0   | 0.9997    | 거의 1 (악성 확신)  |

→ 입력이 아무리 커지거나 작아져도 출력은 항상 0~1 사이. 중간(z=0)에서 가장 가파르다.
이 곡선(비선형성)이 신경망에 "복잡한 패턴을 배우는 힘"을 준다. (직선만 쌓으면 직선밖에 안 됨)

## STEP 2 — 층(layer)과 행렬, 그리고 feedforward

### 가장 중요한 직관 (꼭 기억!)
> **뉴런 1개는 입력 "전부"를 받는다. (1대1 짝짓기가 아니다!)**

입력이 30개면 뉴런 하나는 가중치를 30개 가진다. 그 30개를 전부 받아 z 하나를 만들고, 출력 1개를 낸다.

- 뉴런을 24개 두면 → 출력 24개 = 하나의 "층(layer)"
- **층의 출력 개수 = 뉴런 개수** (이 값은 내가 정한다. PDF의 `--layer 24 24 24`가 이것)
- 입력은 모든 뉴런에 공유, 가중치는 뉴런마다 각자 → 이래서 "fully connected / dense" 층

### 가중치 행렬 W의 크기 (구현 시 버그 1순위!)
```
W 크기 = (뉴런 수, 입력 수)

예: 입력 30개, 뉴런 24개  →  W 크기 = (24, 30),  연결선 = 24×30 = 720개
```

한 층의 계산을 행렬 한 줄로:
```
z = W × 입력 + b
출력 = 활성화(z)

numpy로는:   z = W @ x + b ;  a = sigmoid(z)
```
뉴런이 3개든 100개든 코드는 똑같다. 행렬 크기만 달라질 뿐. → 이래서 선형대수(numpy)가 필수.

### Feedforward (순전파) = 예측하기
한 층의 출력이 다음 층의 입력이 된다. 끝까지 흘리면 예측 완성.

```
입력30 ─[뉴런24]→ 24 ─[뉴런24]→ 24 ─[뉴런2]→ 2
       W(24,30)      W(24,24)      W(2,24)

[세포특징 30개] → 은닉1 → 은닉2 → 출력 → [양성확률, 악성확률]
```
각 화살표가 하는 일은 전부 `활성화(W @ 입력 + b)` 로 동일. 그냥 층마다 반복.
(PDF 예시 구조: 입력 30 → 은닉 24 → 은닉 24 → 출력 2)

## STEP 3 — softmax & loss (얼마나 틀렸나)

### softmax — 출력층 점수를 "확률"로 (PDF 필수 요구)
출력층 뉴런 2개가 점수 2개를 낸다. 그냥 숫자라 해석이 안 됨 → 확률로 변환.

```
softmax: ① 각 점수에 e 씌우고  ② 다 더하고  ③ 각자를 그 합으로 나눈다
```

검증된 예시: 점수 [1.0, 2.0]
```
e^1.0 = 2.71828,  e^2.0 = 7.38906
합 = 10.10734
양성 = 2.71828 / 10.10734 = 0.26894
악성 = 7.38906 / 10.10734 = 0.73106
→ softmax = [0.269, 0.731],  합 = 1.0 ✅ (확률 분포!)
```
왜 e(지수)? ① 모두 양수로 만들고(확률은 음수 불가) ② 점수 차이를 키워 큰 점수를 더 두드러지게.

### one-hot — 정답(label)을 모델 출력과 같은 형식으로
```
양성(B) → [1, 0]      악성(M) → [0, 1]
"정답 자리만 1, 나머지 0"
```

### loss — 틀린 정도를 숫자 하나로 (cross-entropy)
**정답 자리에 모델이 매긴 확률이 1에 가까울수록 loss가 작다. 0에 가까우면 폭발.**

```
loss = -log(정답 자리의 예측 확률)
```
검증 예시: 정답이 악성[0,1]인데 모델이 악성을 0.731로 예측 → loss = -log(0.731) = 0.3133

| 정답 자리 예측 확률 | loss  | 평가          |
|--------------------|-------|---------------|
| 0.99               | 0.01  | 거의 정답      |
| 0.73               | 0.31  | 맞는 방향      |
| 0.50               | 0.69  | 반반           |
| 0.10               | 2.30  | 크게 틀림      |

→ "확신을 갖고 틀리면 크게 벌점". (-log 그래프가 0 근처에서 무한대로 치솟음)

PDF의 binary cross-entropy:  `E = -(1/N) Σ [y·log(p) + (1-y)·log(1-p)]`
→ 모든 샘플에 대해 평균 낸 것. y=1이면 앞항만, y=0이면 뒷항만 살아남음.

### 예측 vs 정답 (헷갈리지 말 것)
```
              자리1(양성)  자리2(악성)
모델 예측:       0.6          0.4      ← softmax 결과(확률). 더 큰 쪽으로 판단 = argmax
정답(one-hot):    1            0      ← 우리가 아는 진짜 답. loss 계산용
```
- 모델의 "판단" = 더 큰 확률 쪽 (첫 자리가 1인지로 보는 게 아님!)
- 맞고 틀림을 아는 이유 = **데이터에 정답이 있어서 = 지도학습(Supervised Learning)**
  - data.csv의 diagnosis 컬럼(M/B)이 정답지. 569명 각각의 정답이 이미 적혀 있음.

## STEP 4 — Gradient Descent (어떻게 고치나)

### 핵심 비유: 안개 낀 산에서 골짜기로 내려가기
발밑밖에 안 보임 → ① 지금 위치의 기울기를 느낌 → ② 내리막 방향으로 한 걸음 → 반복 → 골짜기 바닥(loss 최소) 도착.

### 기울기(gradient) = 미분
> 기울기 = "w를 아주 조금 늘리면, loss가 얼마나·어느 방향으로 변하나?"

| 기울기 부호 | 의미                      | 해야 할 일      |
|------------|---------------------------|----------------|
| 양수 (+)   | w 늘리면 loss 커짐         | w를 줄여야 함   |
| 음수 (-)   | w 늘리면 loss 작아짐       | w를 늘려야 함   |
| 0          | 변화 없음                 | 바닥 도착       |

→ 항상 기울기의 **반대 방향**으로 가면 loss가 줄어든다.

### 업데이트 공식 (외울 것)
```
새 가중치 = 옛 가중치 - (학습률 × 기울기)
```
검증 예시:
```
w=1.0, 학습률=0.2, 기울기=-0.5  →  1.0 - (0.2 × -0.5) = 1.0 + 0.1 = 1.1  (음수기울기→증가 ✅)
w=2.0, 학습률=0.2, 기울기=+3.0  →  2.0 - (0.2 × 3.0)  = 2.0 - 0.6 = 1.4  (양수기울기→감소 ✅)
```
공식의 마이너스 부호가 "기울기 반대 방향 = 내리막"을 자동 처리.

### 학습률(learning rate) = 걸음 크기
- 너무 작으면: 바닥까지 너무 느림
- 너무 크면: 골짜기를 건너뛰어 반대편으로 튕김 → 발산(loss 폭발)
- 적당하면: 효율적으로 수렴
- PDF의 0.0314는 이 데이터에 적당히 튜닝된 값. (직접 조정하는 하이퍼파라미터)

가중치 수천 개를 각자 동시에 업데이트하고, 이걸 epoch만큼 반복 → loss 점점 감소.

## STEP 5 — Backpropagation (기울기를 어떻게 구하나)

### 문제
gradient descent엔 "각 가중치의 기울기"가 필요한데, 가중치가 여러 층에 수천 개 흩어져 있다.
은닉층1 가중치는 loss에 직접이 아니라 은닉2→출력을 거쳐 간접적으로 영향. 이 책임을 어떻게 계산?

### 핵심 비유: 회사의 책임 추궁
프로젝트 실패(loss) → 사장이 책임을 묻는다.
```
사장 ← 부장  ← 과장  ← 사원
(loss) (출력층)(은닉2)(은닉1)
```
사장이 부장에게 책임 배분 → 부장이 과장에게 나눠 전달 → 과장이 사원에게.
**책임이 뒤(출력)에서 앞(입력)으로 거꾸로 흐른다.** = back(뒤로) + propagation(전파).
각자 받은 책임만큼 행동을 고침 = gradient descent 업데이트.

### 왜 거꾸로(출력→입력)인가? ★디펜스 단골 질문★
```
feedforward (예측):   입력 →→→→→→ loss   (앞으로)
backpropagation:      입력 ←←←←←← loss   (뒤로)
```
1. **loss가 신경망 맨 끝(출력)에서 계산된다.**
2. **앞 층 가중치의 책임을 알려면 뒷 층의 책임을 먼저 알아야 한다.**
→ 그래서 출력 끝에서 시작해 입력 쪽으로 거슬러 계산할 수밖에 없다.
+ 뒷 층 계산 결과를 앞 층에서 재활용하니 효율적이다.

### 출발점: 출력층 오차
```
오차 = 예측 - 정답 = [0.27, 0.73] - [0, 1] = [0.27, -0.27]   (정답이 악성 M일 때)
```
이 오차에서 시작해서:
1. 출력층 가중치 기울기 계산 (오차 × 은닉2 출력)
2. 오차를 은닉2로 거꾸로 전파
3. 은닉2 가중치 기울기 계산
4. 은닉1로 전파 → ... 입력층까지
각 단계에서 그 층 가중치의 기울기가 하나씩 나옴. 다 모아서 gradient descent로 일괄 업데이트.

### 왜 미분(연쇄법칙)이 필요한가
"책임을 나누는 비율"을 정확히 계산하는 도구가 미분.
연쇄법칙 한 문장: "A가 B에, B가 C에 영향 → A가 C에 주는 영향 = (A→B) × (B→C)"
→ 영향을 **곱하면서** 뒤로 전파. (수식 디테일은 길 B 문서에서)
지금은 "영향을 곱하며 뒤로 전파한다"만 기억하면 충분.

## 디펜스 대비 — 3대 질문 한 줄 답

- **Feedforward?**
  입력을 층마다 `활성화(W×입력+b)`로 변환하며 출력까지 흘려 예측을 만드는 과정. 데이터가 앞으로만 흐름.

- **Backpropagation?**
  loss가 출력 끝에서 나오므로, 출력에서 입력 쪽으로 거꾸로 연쇄법칙(영향을 곱하며)을 적용해
  각 가중치의 기울기(loss에 대한 책임)를 구하는 과정. 뒷 층 결과를 재활용해 효율적.

- **Gradient descent?**
  `새 w = 옛 w - 학습률×기울기`로 가중치를 기울기 반대(loss 내리막) 방향으로 조금씩 이동시켜
  loss를 최소화하는 방법. 안개 낀 산에서 골짜기로 내려가기.

## 용어 빠른 사전

| 용어              | 뜻                                                    |
|-------------------|-------------------------------------------------------|
| weight (가중치)    | 입력의 중요도. 학습으로 찾는 값                          |
| bias (편향)        | 기준선 보정값                                          |
| activation (활성화) | 가중합에 씌우는 비선형 함수 (sigmoid, softmax 등)        |
| feedforward       | 입력→출력 방향 예측 계산                                 |
| loss              | 예측이 정답과 얼마나 다른지 (cross-entropy)               |
| gradient (기울기)  | w를 조금 바꿀 때 loss 변화량 (= 미분)                    |
| backpropagation   | 기울기를 출력→입력 방향으로 거꾸로 구하는 것              |
| learning rate      | gradient descent 한 걸음의 크기                         |
| epoch             | 전체 학습 루프 1회 반복                                  |
| softmax           | 점수들을 합이 1인 확률로 변환 (출력층)                    |
| one-hot           | 정답 label을 [1,0]/[0,1] 형식으로 표현                   |
| 지도학습           | 정답이 달린 데이터로 학습하는 방식                        |
