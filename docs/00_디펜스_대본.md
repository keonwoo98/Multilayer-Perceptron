# 3. ⭐ 알고리즘 설명 4개 (20점 — 가장 중요)

## 3.1 Feedforward (피드포워드)

**한 줄:** 입력을 층마다 `활성화(W·입력 + b)`로 변환하며 출력까지 흘려 예측을 만드는 과정. 데이터는 앞으로만 흐른다.

**자세히:**
- **뉴런 하나** = 판사. 증거(입력)를 가중치로 저울질해 합치고(가중합 `z = x1·w1 + ... + b`), 활성화 함수로 반응 강도를 정함.
- **층(layer)** = 뉴런 여러 개. 각 뉴런은 **입력을 전부** 받고(1대1 아님), 자기 가중치로 다르게 저울질. → 층 계산 = `활성화(W @ x + b)`, **W 크기 = (뉴런 수, 입력 수)**.
- **쌓기**: 한 층 출력이 다음 층 입력. `30 → 24 → 24 → 2`로 변신.
- **은닉층 sigmoid**(0~1로 누름), **출력층 softmax**(합이 1인 확률 [양성, 악성]).

**코드:** `srcs/layer.py` forward (`self.W @ x + self.b` → 활성화), `srcs/network.py` forward (`for layer in self.layers: x = layer.forward(x)`)

**예상 질문:**
- Q: 왜 활성화 함수가 필요? A: 없으면 층을 쌓아도 하나의 선형변환으로 붕괴. 비선형성이 복잡한 패턴 학습을 가능케 함.
- Q: forward에서 왜 입력을 저장(self.input)? A: backprop에서 가중치 기울기 `dW = 오차 × 입력` 계산에 필요해서.

## 3.2 Gradient descent (경사 하강법)

**한 줄:** loss를 최소화하려고 각 가중치를 기울기 반대 방향으로 조금씩 이동. `새 w = 옛 w − 학습률 × 기울기`.

**비유:** 안개 낀 산에서 발밑 기울기를 느끼고 내리막으로 한 걸음씩 → 골짜기(loss 최소) 도달.

**자세히:**
- 기울기 = "w를 조금 늘리면 loss가 어디로 변하나". 양수면 w 줄이고, 음수면 w 늘림 → 공식의 마이너스가 자동 처리.
- **학습률** = 걸음 크기. 너무 크면 발산, 너무 작으면 느림.
- 우리는 **미니배치**(16개씩 기울기 평균 후 업데이트)를 씀 → 부드럽고 과신 억제.

**코드:** `srcs/optimizer.py` SGD.step (`W -= lr*(dW + wd*W)`), `srcs/network.py` fit (배치 평균 후 optimizer.step)

**예상 질문:**
- Q: batch/stochastic/mini-batch 차이? A: 전체/1개/16개로 기울기 계산. 미니배치는 절충(안정+효율).
- Q: weight_decay(L2)는? A: 기울기에 `wd*W` 더해 가중치를 작게 유지 → 과신/과적합 억제.

## 3.3 Backpropagation (역전파)

**한 줄:** loss가 출력에서 나오니, 출력→입력 방향으로 거꾸로 연쇄법칙을 적용해 각 가중치의 기울기를 구하는 것.

**비유:** 프로젝트 실패(loss) → 사장이 부장→과장→사원으로 책임을 거꾸로 나눠 배분. 각자 받은 책임만큼 행동을 고침.

**자세히:**
- **출발점**: 출력층 오차 = `예측 − 정답`. (softmax+cross-entropy라 미분이 약분돼 뺄셈 하나로 단순화)
- **거꾸로 전파**: 각 은닉 뉴런 오차 = (그 뉴런이 출력에 기여한 가중치 × 출력 오차)의 합 = `W.T @ 오차`, 거기에 sigmoid 미분 곱.
- 전달되는 건 **loss(숫자)가 아니라 오차 신호(벡터)**, 층마다 **값도 크기도 바뀜**(2 → 24 → 24 → 30).
- 각 층 가중치 기울기 `dW = 오차 × 입력`.
- **gradient checking**으로 검증: 수치미분과 1e-10 수준 일치.

**왜 거꾸로?**: ① loss가 출력 끝에서 나옴 ② 앞 층 책임을 알려면 뒷 층 책임을 먼저 알아야 함.

**코드:** `srcs/network.py` backward (`delta = y_pred - y_true; for layer in reversed(self.layers):`), `srcs/layer.py` backward (`dW = outer(delta, input)`, `W.T @ delta * sigmoid'`)

**예상 질문:**
- Q: 왜 출력 오차가 `예측−정답`? A: softmax와 cross-entropy가 짝이라 함께 미분하면 중간 항이 상쇄됨.
- Q: 같은 게 전달되나? A: 아니. 오차 신호가 층마다 재계산돼 값·크기가 바뀜.
- Q: gradient checking? A: 가중치를 ±eps 흔들어 실제 loss 변화(수치 기울기)를 재고 backprop의 dW와 비교. 1e-10 일치 = 정확.

## 3.4 Overfitting (과적합)

**한 줄:** 모델이 학습 데이터를 외워서 train 성능은 좋아지는데 안 본 데이터(validation/test) 성능은 정체·악화되는 상황.

**우리 프로젝트의 생생한 증거 (강력!):**
- 같은 모델로 `data_training.csv` 예측 → BCE **0.03** (외운 데이터, 가짜로 좋음)
- `data_test.csv`(안 본 데이터) 예측 → BCE **0.15** (진짜 실력)
- → 이 **5배 차이가 바로 과적합 개념의 실물**이다.

**어떻게 보나:** 학습곡선에서 train_loss는 내려가는데 val_loss가 벌어지면 과적합.

**우리가 한 해결 (3가지):**
- **L2 정규화(weight_decay)**: 가중치 폭주 억제 → 과신 방지
- **미니배치**: 기울기 평균으로 안정화
- **early stopping**: val_loss 정체 시 중단 + best 가중치 복원

**실제 겪은 사례:** 초기 SGD(샘플 1개씩, 많은 epoch)는 가중치가 커져 과신 → test BCE 0.33 폭발(accuracy는 96%였는데 몇 샘플을 0.000으로 확신하며 틀림). 위 3가지로 ~0.08로 낮춤.

**코드:** `srcs/optimizer.py`(L2), `srcs/network.py` fit(미니배치+early stopping), `srcs/plotting.py`(곡선)

**예상 질문:**
- Q: 과적합 vs 과소적합? A: 과소적합=train·val 둘 다 나쁨(모델 약함). 과적합=train 좋고 val 나쁨(과도 특화).
- Q: accuracy는 높은데 왜 loss 폭발? A: accuracy는 맞/틀만 셈. cross-entropy는 확신한 실수를 강하게 벌함.

# 4. 개념 Q&A (예상 질문 총정리)

### 데이터 관련
- **악성(M) vs 양성(B)?** 악성=암(전이됨, 위험), 양성=암 아님(良性, 순함). "음성(陰性)"은 검사결과 용어로 우리 데이터엔 없음.
- **data.csv 구조?** 569명 × 32컬럼. 1번 id(버림), 2번 diagnosis(정답 M/B), 3~32번 feature 30개(입력).
- **feature 30개 정체?** 세포핵 10가지 측정값(반지름·질감·둘레·면적·매끄러움·조밀도·오목함·오목점·대칭성·프랙탈) × 3버전(mean/se/worst). 코드는 이름 없이 **위치로** 처리해서 이름 몰라도 됨.
- **뭐가 유용한 feature인지 어떻게 알았나?** 직접 안 골랐음. **30개 다 넣고 신경망이 가중치로 스스로 중요도 학습** (유용→큰 가중치, 무용→0 근처). EDA는 우리 이해용.
- **EDA?** 탐색적 데이터 분석. 모델 전에 데이터를 그래프·통계로 살펴 이해. 우리는 분포·스케일·구분력 확인 → 정규화 필요성 발견.

### 전처리 관련
- **정규화 왜?** feature 스케일이 천차만별(area 143~2501 vs smoothness 0.05~0.16). 그냥 넣으면 큰 값이 학습 독점. 표준화로 공평하게.
- **표준화 공식 `(값−평균)/표준편차`?** 통계 표준 기법(z-score). ①평균 빼서 중심 0, ②표준편차로 나눠 퍼진 폭 1. → 모든 feature가 "평균에서 표준편차 몇 칸" 공통 단위.
- **표준편차?** 데이터가 평균에서 얼마나 퍼졌나. `√(평균에서 떨어진 거리²의 평균)`.
- **평균/표준편차는 누구 걸로?** train으로만 계산해 저장, valid/test에도 그 값 적용. (test 걸로 계산하면 정보 누출=cheating)
- **one-hot?** 정답 글자(M/B)를 신경망 출력과 같은 형식으로. B→[1,0], M→[0,1]. 비교 가능하게.

### 분리/평가 흐름 관련
- **train/validation/test 차이?** train=학습용, validation=학습 중 점검(곡선·early stopping), test=최종 채점(안 본 데이터).
- **왜 split이 두 군데(split.py + train_val_split)?** validation은 누설 방지로 학습 데이터에서 와야 함. 우리 워크플로우는 split.py가, 평가(evaluation.py는 validation을 안 만듦)는 train.py가 내부에서 분리. **한 실행엔 하나만 작동.** 분리 로직은 `split_indices` 한 곳 공유.
- **split.py vs evaluation.py?** 둘 다 진짜 분리 프로그램(같은 일). split.py=우리가 만든 제출물("Dataset split" 항목 채점용). evaluation.py=42 제공(공정 채점용, 우리 것 아님, gitignore).
- **무엇을 제출/재생성?** 제출: 코드+data.csv+문서. 재생성(gitignore): 모델(model.npz)·분리데이터·곡선png. 모델은 학습 중 디스크 저장되지만 git엔 안 올림(파생물, 하드코딩 아님 증명).

### 모델/평가 관련
- **가중치 최초값?** 랜덤(×√(2/입력수), He 초기화). 0이면 모든 뉴런 똑같아져 학습 불가(대칭 문제). He는 신호가 층 지나도 적당히 유지되게 스케일 조정. bias는 0(대칭은 랜덤 W가 이미 깸).
- **He 초기화?** `랜덤 × √(2/입력개수)`. 입력 많으면 가중합 폭발 → 입력수로 스케일 줄여 sigmoid 포화 방지. 깊은 망도 학습됨.
- **bias도 학습되나?** 네. 0에서 시작, 매 미니배치마다 W와 함께 `db`로 업데이트.
- **BCE(Binary Cross-Entropy)?** 둘 중 하나 맞히는 문제의 loss. 정답 자리 확률에 −log, 전체 평균. 낮을수록 좋음(0=완벽), 확신하며 틀리면 폭발. predict.py가 출력.
- **왜 accuracy 안 쓰고 BCE?** accuracy는 맞/틀만 셈. BCE는 "얼마나 확신했나"까지 봄(0.51로 맞힘 vs 0.99로 맞힘 구분). 더 정밀.
- **왜 test 점수가 train보다 높나(나쁜가)?** train은 학습한 데이터(외움)라 낮고, test는 안 본 데이터라 높음. test가 진짜 실력 = 과적합 개념.
- **5점이 안 나오는데?** 성능은 split 운(약 50%). 어려운 split은 4점 천장. 모델은 이미 최선. 4점도 좋은 점수.
