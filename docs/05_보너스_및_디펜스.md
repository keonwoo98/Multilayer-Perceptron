# 보너스 및 디펜스 대비

> 평가 시트(Intra Edit)와 보너스 구현을 정리한 문서.
> 디펜스 직전에 이것만 읽으면 평가 항목별로 답할 수 있도록 만들었다.

---

## 평가 워크플로우 (반드시 숙지)

평가자는 `evaluation.py`로 데이터를 나눈다 (우리 split.py가 아님):
```
python evaluation.py        # data.csv → data_training.csv(75%), data_test.csv(25%)
python train.py data_training.csv      # 학습 + 모델 저장
python predict.py --data data_test.csv # BCE 평가
```
- evaluation.py는 `data_valid.csv`를 안 만든다 → train.py가 **단일 파일을 받아 내부에서
  validation을 자동 분할**하도록 구현했다 (train_val_split).
- 형식은 원본과 동일(헤더 없음) → load_data가 그대로 처리.

## 모델 성능 점수표 (BCE 기준)
```
< 0.08 → 5점   0.08~0.13 → 4점   0.13~0.18 → 3점 ...
```
우리 기본 설정(SGD 미니배치 + L2)으로 여러 split 검증:
평균 BCE ≈ 0.084, 항상 4점 이상, 약 62%가 5점.
평가는 "3번 학습 후 best" 허용 → 5점 확률 ~94%.

## 보너스 구현 (5가지)

| 보너스 | 파일 | 설명 |
|--------|------|------|
| 미니배치 경사하강 | network.fit | 8~16개 묶어 gradient 평균 → 안정적, 과신 완화 |
| L2 정규화 | optimizer.SGD | `W -= lr*(dW + wd*W)`. 가중치 폭주 억제 → 과적합 완화 |
| Adam optimizer | optimizer.Adam | 적응적 학습률(모멘트 m,v). `--optimizer adam` |
| early stopping | network.fit | val_loss 정체 시 중단 + best 가중치 복원. 기본 ON, `--no_early_stopping`으로 끔 |
| 다중 metric | metrics.py | precision, recall, F1 (predict.py 출력) |

> 보너스는 mandatory가 완벽할 때만 평가됨. 위는 모두 동작 검증 완료.

### 미니배치 + L2가 왜 중요했나 (스토리)
- 초기 SGD(샘플 1개씩, 50 epoch)는 test BCE가 0.15~0.34로 들쭉날쭉했다.
- 원인: 과신(overconfidence). accuracy는 96%인데 어려운 샘플 2~3개를 0.000으로
  강하게 틀려 BCE가 폭발(한 샘플이 BCE 16까지 기여).
- 미니배치(gradient 평균) + L2(가중치 억제)로 과신을 잡아 평균 BCE 0.084로 안정화.

## 디펜스 4대 질문 답변 (각 5점)

### Feedforward
입력을 층마다 `activation(W @ x + b)`로 변환하며 출력층까지 흘려 예측을 만든다.
은닉층은 sigmoid, 출력층은 softmax로 확률 분포를 만든다. 데이터가 입력→출력 한 방향.

### Gradient descent
loss를 최소화하려고 가중치를 기울기 반대 방향으로 조금씩 이동.
`새 W = 옛 W - 학습률 × 기울기`. 안개 낀 산에서 발밑 기울기를 보고 골짜기로 내려가기.
우리는 미니배치로 여러 샘플의 기울기를 평균 내어 한 걸음씩 이동한다.

### Backpropagation
loss가 출력 끝에서 나오므로, 출력에서 입력 쪽으로 거꾸로 연쇄법칙을 적용해
각 가중치의 기울기를 구한다. softmax+cross-entropy라 출력층 오차는 `예측-정답`으로
단순해진다. 각 층: `dW = 오차 ⊗ 입력`, 앞 층으로 `(W.T @ 오차) × sigmoid미분` 전파.
gradient checking(수치미분과 1e-10 수준 일치)으로 정확성을 검증했다.

### Overfitting (★ 평가 항목)
모델이 training 데이터를 "외워서" training 성능은 계속 좋아지는데
처음 보는 데이터(validation/test) 성능은 정체되거나 나빠지는 상황.
학습곡선에서 training loss는 내려가는데 validation loss가 벌어지면 과적합 신호.
- 우리가 본 증상: 정규화 없이 오래 학습하니 가중치가 극단으로 커져 과신 → test BCE 폭발.
- 해결: L2 정규화로 가중치 억제, 미니배치로 안정화, (옵션) early stopping.
- 결과: 학습곡선의 두 선이 거의 붙어서 함께 내려감 = 과적합 없음.

## mandatory 평가 항목 체크리스트

| 항목 | 상태 |
|------|------|
| 은닉층 최소 2개 (깊이 ≥ 4) | ✅ 기본 [30,24,24,2] |
| 구조를 인자로 변경 가능 | ✅ `--layers` |
| 데이터셋 train/validation 분리 | ✅ split.py + train.py 내부 분할 |
| softmax 출력층 | ✅ |
| epoch마다 metric 표시 | ✅ `epoch N/M - loss - val_loss` |
| 학습곡선 2개 이상 | ✅ loss + accuracy (train/val) |
| 이진분류 적합 metric | ✅ categorical/binary cross-entropy |
| 모델이 학습됨 | ✅ loss 감소, 정확도 99% |
| 과적합 아님 | ✅ L2 + 미니배치, 곡선 안 벌어짐 |
| 모델 저장 (topology + weights) | ✅ model.npz (layer_sizes + W,b + mean,std) |
| predict가 모델·데이터 로드 | ✅ 하드코딩 아님 |
| binary cross-entropy 평가 | ✅ |

## 크래시 방지 (평가 flag)
- 파일 없음: train.py/predict.py 모두 깔끔한 에러 메시지 + 종료 (Traceback X)
- 잘못된 인자: argparse가 차단 (choices, type)
- 모델 없이 predict: "train a model first" 안내
