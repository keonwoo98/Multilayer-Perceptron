# 데이터셋 분석 (data.csv)

> PDF가 강조한 "데이터 이해 단계(data understanding phase)"의 결과물.
> Wisconsin Breast Cancer 데이터셋. 학습 시작 전에 데이터를 그래프로 보고 이해한다.

---

## 1. 기본 구조

```
총 샘플(행): 569개
총 컬럼: 32개
결측치: 없음
헤더: 없음 (csv 첫 줄부터 데이터)
```

| 컬럼 위치 | 이름        | 역할                                              |
|-----------|-------------|---------------------------------------------------|
| 1번       | id          | 환자 식별번호 → **학습에서 제외** (진단과 무관)     |
| 2번       | diagnosis   | **정답(label)**: M(악성) 또는 B(양성)              |
| 3~32번    | feature 30개 | 세포핵 특징 (입력)                                |

### 30개 feature의 구조
사실 **10개 측정값 × 3종류(mean / se / worst)** = 30개다.

10개 측정값: radius(반지름), texture(질감), perimeter(둘레), area(면적),
smoothness(매끄러움), compactness(조밀도), concavity(오목함),
concave_points(오목점 개수), symmetry(대칭성), fractal_dim(프랙탈 차원)

- `_mean`  = 평균값
- `_se`    = 표준오차(standard error)
- `_worst` = 가장 나쁜(큰) 값들의 평균

> 주의: data.csv에는 컬럼명이 없으므로, 코드에서 직접 이 순서로 이름을 붙여야 한다.

## 2. label 분포 (약간 불균형)

```
B (양성, Benign):    357개  (62.7%)
M (악성, Malignant): 212개  (37.3%)
```
- 약 6:4. 학습에 문제없는 수준. (99:1 정도였다면 대책이 필요했을 것)
- train/validation 분리 시, 이 비율이 양쪽에 고르게 유지되도록 하면 더 좋다(stratify).

## 3. ★ 정규화가 필요한 결정적 증거 — `images/04_normalization.png`

feature마다 값의 스케일이 천차만별이다 (왼쪽 그래프, 로그 스케일):

```
area_mean       범위: 143.5 ~ 2501.0     ← 수백~수천
smoothness_mean 범위: 0.0526 ~ 0.1634    ← 0.0x
```

→ area는 smoothness보다 1만 배 이상 크다.
이대로 신경망에 넣으면 **값이 큰 feature(area)가 학습을 부당하게 지배**한다.

**해결: 표준화(standardization)**
```
표준화된 값 = (값 - 그 feature의 평균) / 그 feature의 표준편차
```
오른쪽 그래프처럼 모든 feature가 평균 0, 표준편차 1 근처로 맞춰진다 → 공평하게 학습에 기여.

**중요 구현 규칙:**
- 평균/표준편차는 **training 데이터로만** 계산한다.
- 그 값을 validation/test에도 **그대로** 적용한다. (test 데이터로 평균 내면 정보 누출=cheating)
- 그래서 평균/표준편차 값을 모델과 함께 저장해야 predict 때 동일 변환 가능.

## 4. 어떤 feature가 악성/양성을 잘 구분하나 — `images/01_feature_distributions.png`

악성(M)이 양성(B)보다 평균이 큰 정도 (배율 M/B, 큰 순):

| feature             | B 평균  | M 평균  | 배율 M/B |
|---------------------|---------|---------|----------|
| concavity_mean      | 0.046   | 0.161   | 3.49     |
| concave_points_mean | 0.026   | 0.088   | 3.42     |
| area_mean           | 462.8   | 978.4   | 2.11     |
| compactness_mean    | 0.080   | 0.145   | 1.81     |
| perimeter_mean      | 78.1    | 115.4   | 1.48     |
| radius_mean         | 12.1    | 17.5    | 1.44     |
| texture_mean        | 17.9    | 21.6    | 1.21     |
| smoothness_mean     | 0.092   | 0.103   | 1.11     |
| symmetry_mean       | 0.174   | 0.193   | 1.11     |
| fractal_dim_mean    | 0.063   | 0.063   | 1.00     |

해석:
- **악성 세포는 더 크고(area/radius/perimeter), 더 울퉁불퉁(concavity/concave_points)** 하다. 직관과 일치.
- 히스토그램에서 concavity, concave_points, radius, perimeter, area는 파란(B)/빨강(M) 봉우리가
  뚜렷이 갈라진다 → 분류에 유용한 feature.
- 반대로 **fractal_dim_mean(배율 1.00), symmetry, smoothness**는 두 분포가 거의 겹친다 → 구분력 약함.

## 5. feature 간 상관관계 — `images/02_correlation.png`

- **radius ↔ perimeter ↔ area = 상관계수 0.99~1.00** → 사실상 같은 정보(크기). 당연하다(둘레·면적은 반지름의 함수).
- concavity ↔ concave_points = 0.92, compactness ↔ concavity = 0.88 → 모양 관련끼리 묶임.
- texture, fractal_dim은 다른 것들과 상관이 낮아 독립적 정보를 준다.

함의: 중복 정보가 많다. 신경망은 알아서 가중치로 처리하지만,
보너스나 실험에서 feature를 줄여도 성능이 크게 안 떨어질 수 있다는 힌트.

## 6. 두 feature만으로도 잘 갈린다 — `images/03_scatter.png`

가장 구분력 좋은 두 feature(concave_points_mean, radius_mean)로 산점도를 그리면
양성(파랑, 왼쪽 아래)과 악성(빨강, 오른쪽 위)이 비교적 깔끔하게 분리된다.
→ 이 데이터는 선형에 가까운 패턴도 상당해서, 잘 만든 MLP면 높은 정확도(~97%+)가 나온다.

## 7. 전처리 체크리스트 (구현 시)

```
1. csv 로드 + 컬럼명 부여 (헤더 없음 주의)
2. id 컬럼 제거
3. diagnosis(M/B)를 숫자로:  B→0, M→1  (그리고 one-hot: B→[1,0], M→[0,1])
4. feature 30개 표준화 (평균/표준편차는 train으로만 계산, 값 저장)
5. train/validation 분리 (seed 고정, 가능하면 M/B 비율 유지)
```

생성된 그래프: `docs/images/01~04_*.png`
재현 스크립트는 추후 `visualize.py`(또는 EDA 노트북)로 정리 예정.
