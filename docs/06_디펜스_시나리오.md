# 1. Modularity / 모듈러성

### What the evaluator checks / 평가자가 확인하는 것
- At least 2 hidden layers by default (total depth ≥ 4 with input+output).
- The number of hidden layers can be changed via file or arguments.
- 기본 은닉층 2개 이상(입력·출력 포함 깊이 ≥ 4), 인자로 층 수 변경 가능.

### Core answer / 핵심 답변
**EN:** "By default the network is `[30, 24, 24, 2]` — input 30, two hidden layers of 24, output 2. That's depth 4. The architecture is fully driven by the `--layers` argument in `train.py`: I pass the hidden-layer sizes and the code prepends the input size (30) and appends the output size (2). `Network.__init__` then builds one `DenseLayer` per consecutive pair, so I can make it deeper or wider without touching the code."

**KO:** "기본 구조는 `[30, 24, 24, 2]`로 입력 30, 은닉층 24 두 개, 출력 2이며 깊이 4입니다. 구조는 `train.py`의 `--layers` 인자로 완전히 제어됩니다. 은닉층 크기만 주면 코드가 앞에 입력 30, 뒤에 출력 2를 붙이고, `Network.__init__`이 인접한 크기 쌍마다 `DenseLayer`를 만들어 쌓습니다. 코드 수정 없이 더 깊거나 넓게 만들 수 있습니다."

### Code evidence / 코드 근거
- `train.py` — argparse builds the full topology:
```python

...
```

# 3. Softmax / 소프트맥스

### What the evaluator checks / 평가자가 확인하는 것
- Softmax is implemented on the output layer and turns the output into a probability distribution.
- 출력층에 softmax가 구현되어 출력을 확률 분포로 만든다.

### Core answer / 핵심 답변
**EN:** "The output layer uses softmax — `Network.__init__` sets the last layer's activation to `'softmax'`, every hidden layer to `'sigmoid'`. Softmax exponentiates each score and divides by the sum, so the two outputs are positive and add to 1: `[P(benign), P(malignant)]`. I subtract the max before the exponential for numerical stability, which doesn't change the result."

**KO:** "출력층은 softmax를 씁니다. `Network.__init__`이 마지막 층의 활성화를 `'softmax'`로, 은닉층은 모두 `'sigmoid'`로 설정합니다. softmax는 각 점수에 지수를 취해 합으로 나누므로 두 출력이 모두 양수이고 합이 1이 됩니다: `[P(양성), P(악성)]`. 지수 전에 최댓값을 빼서 수치 안정성을 확보하며 결과는 동일합니다."

### Code evidence / 코드 근거
- `srcs/activations.py`:
```python
def softmax(z):
    z_shift = z - np.max(z)      # numerical stability, result unchanged
    exp = np.exp(z_shift)
    return exp / np.sum(exp)
```
- `srcs/network.py` — last layer is softmax (see `is_last` in `__init__`).

### Live demo / 라이브 데모
```bash
python -c "import numpy as np; from srcs.activations import softmax; \
print(softmax(np.array([1.0,2.0])), softmax(np.array([1.0,2.0])).sum())"
# [0.2689 0.7311] 1.0
```

### Likely follow-ups / 예상 추가 질문
- **Q (EN):** Why subtract the max? **A:** `exp` of a large score overflows; subtracting the max keeps exponents ≤ 0. Numerator and denominator shrink by the same factor, so probabilities are identical.
  **Q (KO):** 왜 최댓값을 빼나? **A:** 큰 점수의 `exp`는 오버플로우가 나는데, 최댓값을 빼면 지수가 0 이하가 됩니다. 분자·분모가 같은 비율로 줄어 확률은 동일합니다.
- **Q (EN):** Why softmax instead of one sigmoid output? **A:** The subject asks for a probabilistic distribution and 2 output neurons; softmax gives a normalized distribution over the two classes and pairs cleanly with categorical cross-entropy.
  **Q (KO):** 왜 sigmoid 하나가 아니라 softmax인가? **A:** 서브젝트가 확률 분포와 출력 뉴런 2개를 요구하며, softmax는 두 클래스에 대한 정규화 분포를 주고 categorical cross-entropy와 잘 맞습니다.
- **Q (EN):** How do you turn the distribution into a class? **A:** `argmax` over the two probabilities (column 1 = malignant).
  **Q (KO):** 분포를 클래스로 어떻게 바꾸나? **A:** 두 확률의 `argmax`(1열 = 악성)로 정합니다.

# 6. Model learns and is not overfitting / 학습 및 과적합 없음

### What the evaluator checks / 평가자가 확인하는 것
- The model learns (train and validation metrics improve over epochs).
- The model is not overfitting.
- 모델이 학습되고(지표가 에폭에 따라 개선), 과적합이 아님.

### Core answer / 핵심 답변
**EN:** "Loss drops from ~0.6 at epoch 1 to ~0.04 by the end, and accuracy reaches ~98–99% on both train and validation, so it clearly learns. It's not overfitting because the training and validation curves stay close together and the validation loss doesn't diverge upward. I get that behaviour through three things: mini-batch gradient averaging (smoother updates), L2 weight decay (`weight_decay=1e-3`, keeps weights small), and early stopping which is ON by default (restores the best-validation weights; disable with `--no_early_stopping`)."

**KO:** "손실이 1에폭의 약 0.6에서 마지막에 약 0.04로 떨어지고 정확도가 학습·검증 모두 약 98~99%에 도달하므로 분명히 학습합니다. 학습·검증 곡선이 가깝게 붙어 있고 검증 손실이 위로 발산하지 않으므로 과적합이 아닙니다. 이는 세 가지로 달성합니다: 미니배치 기울기 평균(부드러운 갱신), L2 가중치 감쇠(`weight_decay=1e-3`, 가중치를 작게 유지), 그리고 기본으로 켜진 조기 종료(best 검증 가중치 복원, `--no_early_stopping`으로 끔)."

### Code evidence / 코드 근거
- `srcs/optimizer.py` `SGD.step` — L2 term:
```python
layer.W -= self.lr * (dW + self.weight_decay * layer.W)
```
- `srcs/network.py` `fit` — mini-batch averaging before each update; optional early stopping with best-weight restore.
- `srcs/layer.py` — He init keeps signal healthy so training is stable.

### Live demo / 라이브 데모
```bash

# the two loss lines descend together and stay close -> no overfitting
```

### Likely follow-ups / 예상 추가 질문
- **Q (EN):** How would you SEE overfitting if it happened? **A:** Training loss keeps dropping while validation loss flattens then rises — the two curves separate. With `--weight_decay 0` and a high `lr` you can reproduce that.
  **Q (KO):** 과적합을 어떻게 눈으로 보나? **A:** 학습 손실은 계속 내려가는데 검증 손실이 정체 후 상승하며 두 곡선이 벌어집니다. `--weight_decay 0`에 높은 `lr`로 재현 가능합니다.
- **Q (EN):** Why does L2 reduce overfitting? **A:** It penalizes large weights, so the model can't become over-confident; predictions stay less extreme and generalize better.
  **Q (KO):** L2가 왜 과적합을 줄이나? **A:** 큰 가중치에 페널티를 줘 모델이 과신하지 못하게 하므로 예측이 덜 극단적이고 일반화가 좋아집니다.
- **Q (EN):** Does mini-batch help generalization? **A:** Averaging gradients over a batch smooths the updates, reducing the noisy, over-confident jumps that single-sample SGD produced.
  **Q (KO):** 미니배치가 일반화에 도움? **A:** 배치 평균이 갱신을 부드럽게 해, 샘플 1개 SGD의 잡음 많은 과신 점프를 줄입니다.

# 10. Algorithm: Feedforward / 피드포워드

### What the evaluator checks / 평가자가 확인하는 것
- Explain the feedforward algorithm (rated 0–5).

### Core answer / 핵심 답변
**EN:** "Feedforward is how the network turns an input into a prediction. Each layer computes `z = W @ x + b` (a weighted sum of all its inputs plus a bias) and applies an activation: sigmoid for hidden layers, softmax for the output. The output of one layer is the input to the next, so `Network.forward` just loops the layers in order. The final output is `[P(benign), P(malignant)]`. Data only flows forward, input to output."

**KO:** "피드포워드는 입력을 예측으로 바꾸는 과정입니다. 각 층이 `z = W @ x + b`(모든 입력의 가중합 + 편향)를 계산하고 활성화를 적용합니다 — 은닉층은 sigmoid, 출력층은 softmax. 한 층의 출력이 다음 층의 입력이 되므로 `Network.forward`는 층을 순서대로 반복합니다. 최종 출력은 `[P(양성), P(악성)]`이고, 데이터는 입력에서 출력으로만 흐릅니다."

### Code evidence / 코드 근거
- `srcs/layer.py` `forward`:
```python
self.input = x                  # cached for backprop
z = self.W @ x + self.b
return sigmoid(z)   # or softmax(z) on the output layer
```
- `srcs/network.py` `forward`:
```python
for layer in self.layers:
    x = layer.forward(x)
return x
```

- **Q (EN):** Why store `self.input` during forward? **A:** Backprop needs the layer's input to build the weight gradient `dW = outer(delta, input)`, so forward caches it.
  **Q (KO):** forward에서 왜 `self.input` 저장? **A:** 역전파가 가중치 기울기 `dW = outer(delta, input)`를 만들 때 그 층의 입력이 필요해서 캐시합니다.
- **Q (EN):** What's the shape of `W`? **A:** `(n_neurons, n_inputs)` — e.g. the first hidden layer is `(24, 30)`.
  **Q (KO):** `W`의 형태는? **A:** `(뉴런 수, 입력 수)` — 첫 은닉층은 `(24, 30)`.
- **Q (EN):** Why activations at all? **A:** Without a non-linear activation, stacked linear layers collapse into one linear map; sigmoid/softmax give the network its expressive power.
  **Q (KO):** 활성화가 왜 필요? **A:** 비선형 활성화가 없으면 쌓인 선형 층이 하나의 선형 변환으로 붕괴합니다. sigmoid/softmax가 표현력을 줍니다.

# 11. Algorithm: Gradient descent / 경사 하강법

### What the evaluator checks / 평가자가 확인하는 것
- Explain the gradient descent algorithm (rated 0–5).

### Core answer / 핵심 답변
**EN:** "Gradient descent minimizes the loss by nudging each weight in the opposite direction of its gradient: `W = W - learning_rate * gradient`. The gradient says how the loss changes if a weight increases; moving against it lowers the loss. I use mini-batch gradient descent: each step averages the gradients over a batch of 16 samples and then the optimizer updates the weights. The learning rate is the step size — too big diverges, too small is slow."

**KO:** "경사 하강법은 각 가중치를 기울기의 반대 방향으로 조금씩 움직여 손실을 최소화합니다: `W = W - 학습률 * 기울기`. 기울기는 가중치를 키울 때 손실이 어떻게 변하는지를 말하며, 반대로 움직이면 손실이 줄어듭니다. 저는 미니배치 경사 하강법을 씁니다: 각 스텝이 16개 샘플의 기울기를 평균낸 뒤 옵티마이저가 가중치를 갱신합니다. 학습률은 보폭으로, 너무 크면 발산하고 너무 작으면 느립니다."

### Code evidence / 코드 근거
- `srcs/optimizer.py` `SGD.step`:
```python
layer.W -= self.lr * (dW + self.weight_decay * layer.W)
layer.b -= self.lr * db
```
- `srcs/network.py` `fit` — average batch gradients, then `optimizer.step`:
```python
grads = [(gW / m, gb / m) for gW, gb in grads]   # m = batch size
optimizer.step(self.layers, grads)
```

- **Q (EN):** Batch vs stochastic vs mini-batch? **A:** Full-batch uses all samples per step (smooth, slow), stochastic uses 1 (noisy), mini-batch (16) balances both — what I use.
  **Q (KO):** 배치/확률적/미니배치 차이? **A:** 풀배치는 매 스텝 전체(부드럽지만 느림), 확률적은 1개(잡음), 미니배치(16)는 절충이며 제가 쓰는 방식입니다.

# 12. Algorithm: Backpropagation / 역전파

### What the evaluator checks / 평가자가 확인하는 것
- Explain the backpropagation algorithm (rated 0–5).

### Core answer / 핵심 답변
**EN:** "Backpropagation computes each weight's gradient efficiently using the chain rule, going from the output back to the input. Because I use softmax + cross-entropy, the output-layer error simplifies to `delta = prediction - target`. For each layer, going backwards: the weight gradient is `dW = outer(delta, layer_input)`, and the error passed to the previous layer is `(W.T @ delta) * sigmoid'(its output)`. I verified it with gradient checking — my backprop matches a numerical gradient to about 1e-10."

**KO:** "역전파는 연쇄 법칙으로 각 가중치의 기울기를 출력에서 입력 방향으로 효율적으로 계산합니다. softmax + 교차 엔트로피를 쓰므로 출력층 오차가 `delta = 예측 - 정답`으로 단순해집니다. 각 층을 거꾸로 가며: 가중치 기울기는 `dW = outer(delta, 층 입력)`, 이전 층으로 전달되는 오차는 `(W.T @ delta) * sigmoid'(그 출력)`입니다. gradient checking으로 검증했고, 제 역전파가 수치 기울기와 약 1e-10까지 일치합니다."

### Code evidence / 코드 근거
- `srcs/network.py` `backward`:
```python
delta = y_pred - y_true                 # softmax + cross-entropy => clean start
for layer in reversed(self.layers):
    delta = layer.backward(delta)
```
- `srcs/layer.py` `backward`:
```python
self.dW = np.outer(delta, self.input)
self.db = delta
da_prev = self.W.T @ delta
return da_prev * sigmoid_derivative(self.input)
```

### Live demo / 라이브 데모
```bash

python - << 'PY'

from srcs.network import Network
from srcs.losses import categorical_cross_entropy
net=Network([4,3,2],seed=1); x=np.array([.5,-.3,.8,.1]); y=np.array([0,1])
net.backward(net.forward(x),y); eps=1e-6
def L():p=net.forward(x);return categorical_cross_entropy(p.reshape(1,-1),y.reshape(1,-1))
m=0
for ly in net.layers:
  for i in range(ly.W.shape[0]):
    for j in range(ly.W.shape[1]):
      o=ly.W[i,j];ly.W[i,j]=o+eps;a=L();ly.W[i,j]=o-eps;b=L();ly.W[i,j]=o
      m=max(m,abs(ly.dW[i,j]-(a-b)/(2*eps)))
print("max diff:", m)   # ~1.7e-10
PY
```

### Likely follow-ups / 예상 추가 질문
- **Q (EN):** Why does the output error equal `prediction - target`? **A:** When softmax is paired with cross-entropy, the intermediate derivatives cancel and the gradient of the loss w.r.t. the pre-softmax scores is exactly `prediction - target`.
  **Q (KO):** 왜 출력 오차가 `예측 - 정답`인가? **A:** softmax와 교차 엔트로피가 짝을 이루면 중간 미분이 상쇄되어, softmax 이전 점수에 대한 손실의 기울기가 정확히 `예측 - 정답`이 됩니다.
- **Q (EN):** What is `sigmoid'`? **A:** `sigmoid'(z) = a * (1 - a)` where `a = sigmoid(z)`; I reuse the cached output, so it's `self.input * (1 - self.input)` for the previous layer.
  **Q (KO):** `sigmoid'`는? **A:** `a = sigmoid(z)`일 때 `sigmoid'(z) = a*(1-a)`이며, 캐시된 출력을 재사용해 이전 층에 대해 `self.input*(1-self.input)`입니다.
- **Q (EN):** Why reversed order? **A:** Each layer's gradient depends on the error coming from the layer after it, so we must compute from the output backward.
  **Q (KO):** 왜 역순인가? **A:** 각 층의 기울기는 뒤 층에서 오는 오차에 의존하므로 출력에서 거꾸로 계산해야 합니다.
- **Q (EN):** What is gradient checking? **A:** Perturb a weight by ±eps, measure the actual loss change (numerical gradient), and compare to backprop's `dW`. Matching to ~1e-10 proves backprop is correct.
  **Q (KO):** gradient checking이란? **A:** 가중치를 ±eps 흔들어 실제 손실 변화(수치 기울기)를 재고 역전파의 `dW`와 비교합니다. ~1e-10 일치는 역전파가 옳다는 증거입니다.

# 13. Algorithm: Overfitting / 과적합

### What the evaluator checks / 평가자가 확인하는 것
- Explain what a situation of overfitting is (rated 0–5).

### Core answer / 핵심 답변
**EN:** "Overfitting is when the model memorizes the training data instead of learning the general pattern: training loss keeps improving but validation/test performance stalls or gets worse. You see it as the training and validation curves separating. In my project I hit a concrete case — plain single-sample SGD trained for many epochs made the weights huge and the model over-confident; accuracy was ~96% but test BCE blew up to 0.33 because a few confidently-wrong samples each contributed a huge loss. I fixed it with mini-batch averaging, L2 weight decay, and optional early stopping, bringing test BCE down to ~0.03–0.08 with the curves staying together."

**KO:** "과적합은 모델이 일반 패턴을 배우는 대신 학습 데이터를 외우는 상황입니다: 학습 손실은 계속 좋아지는데 검증/테스트 성능은 정체되거나 나빠집니다. 학습·검증 곡선이 벌어지는 것으로 보입니다. 제 프로젝트에서 구체적 사례를 겪었습니다 — 샘플 1개 SGD로 오래 학습하니 가중치가 커지고 모델이 과신했습니다. 정확도는 약 96%였지만 확신하며 틀린 몇 샘플이 각각 큰 손실을 내 test BCE가 0.33까지 폭발했습니다. 미니배치 평균, L2 가중치 감쇠, 선택적 조기 종료로 고쳐 test BCE를 약 0.03~0.08로 낮추고 곡선이 붙게 했습니다."

### Likely follow-ups / 예상 추가 질문
- **Q (EN):** How is overfitting different from underfitting? **A:** Underfitting = high error on both train and val (model too weak); overfitting = low train error but high val error (model too specialized).
  **Q (KO):** 과적합 vs 과소적합? **A:** 과소적합은 학습·검증 둘 다 오차 높음(모델이 약함), 과적합은 학습 오차는 낮은데 검증 오차가 높음(모델이 과도하게 특화).
- **Q (EN):** Why did accuracy stay high while loss blew up? **A:** Accuracy only counts right/wrong; cross-entropy punishes confident mistakes heavily, so a few `p≈0` on a true-malignant sample dominate the average.
  **Q (KO):** 정확도는 높은데 왜 손실이 폭발? **A:** 정확도는 맞/틀만 세지만, 교차 엔트로피는 확신한 실수를 강하게 벌합니다. 악성 정답에 `p≈0` 몇 개가 평균을 지배합니다.
- **Q (EN):** Name three anti-overfitting tools you used. **A:** L2 weight decay, mini-batch gradient averaging, early stopping (plus a validation set to detect it).
  **Q (KO):** 과적합 방지 도구 3가지? **A:** L2 가중치 감쇠, 미니배치 기울기 평균, 조기 종료(그리고 탐지용 검증셋).

### Likely follow-ups / 예상 추가 질문
- **Q (EN):** Explain Adam briefly. **A:** It keeps a running mean (`m`) and variance (`v`) of the gradient per parameter, bias-corrects them, and steps by `lr * m_hat / (sqrt(v_hat) + eps)` — adaptive per-weight step sizes.
  **Q (KO):** Adam을 간단히? **A:** 파라미터별 기울기의 이동평균(`m`)과 분산(`v`)을 유지·편향보정하고 `lr * m_hat / (sqrt(v_hat)+eps)`로 갱신합니다 — 가중치별 적응적 보폭.
- **Q (EN):** Why did SGD beat Adam here? **A:** On this small dataset, well-tuned SGD + L2 generalized slightly better; Adam at lr 0.001 was fine but not better, so SGD is the default.
  **Q (KO):** 왜 SGD가 Adam보다 좋았나? **A:** 이 작은 데이터셋에서 잘 튜닝된 SGD+L2가 약간 더 잘 일반화했고, lr 0.001의 Adam도 괜찮지만 더 낫진 않아 SGD가 기본입니다.
- **Q (EN):** What does precision vs recall mean here? **A:** Precision = of those predicted malignant, how many truly are; recall = of truly malignant, how many we caught. For cancer, recall matters a lot.
  **Q (KO):** 정밀도와 재현율 의미? **A:** 정밀도 = 악성이라 예측한 것 중 실제 악성 비율; 재현율 = 실제 악성 중 잡아낸 비율. 암에서는 재현율이 특히 중요합니다.
